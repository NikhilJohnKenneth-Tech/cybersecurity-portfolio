# Day 1 – OSI Model

## Objective
Understand the OSI model and how data flows through network layers.

## What I Learned
- The OSI model has 7 layers:
  1. Physical
  2. Data Link
  3. Network
  4. Transport
  5. Session
  6. Presentation
  7. Application

## Notes
- Each layer has a specific role in communication.
- Data moves from Application → Physical when sending.
- Data moves from Physical → Application when receiving.

## Next Steps
- Study TCP/IP model
- Map real-world example (browser request) to OSI layers

## Professional Understanding

-The OSI model is not just theoretical. It helps identify where communication breaks or where an attack occurs.
-Each layer has specific responsibilities, and analyzing issues layer-by-layer improves troubleshooting and security analysis.
